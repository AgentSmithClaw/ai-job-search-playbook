# KMS 使用说明

> 这是 `ai-job-search-playbook` 内部的个人知识管理系统最小使用指南。

## 当前组成
- 系统说明：`pkm-system.md`
- 执行流程：`pkm-workflow.md`
- 模板包：`pkm-templates.md`
- 标签规则：`pkm-tagging-rules.md`
- 项目进度：`../06-logs/kms-progress.md`
- Inbox：`../00-Inbox/`
- Daily Notes：`../06-logs/daily-notes/`

## 每天怎么用
1. 临时信息先放进 `00-Inbox/`
2. 每天至少写一份 Daily Note
3. 把当天最重要的 3 条知识点写下来
4. 若有可复用内容，转移到正式模块文档

## 每周怎么用
1. 清理 Inbox
2. 做一次 Weekly Review
3. 把零散记录提炼成项目经验 / 面试表达 / 学习笔记
4. 给高价值内容补充标签与关联文档

## 每月怎么用
1. 做一次 Monthly Review
2. 调整分类和标签
3. 清理低价值内容
4. 确认哪些知识已经转化成实际输出

## 当前建议
- 主承载先用当前 Markdown 仓库，成本最低
- 等内容规模起来，再决定是否同步到 Notion 或 Obsidian
- 不追求一开始就完美，先保证记录和整理持续发生
