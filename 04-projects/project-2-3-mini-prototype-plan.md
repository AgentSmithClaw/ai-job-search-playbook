# 项目2 / 项目3 最小可展示样例计划

## 目标
把 `ai-research-brief-generator` 和 `ai-workflow-opportunity-scanner` 从“讲得通”推进到“至少有一个看得见的轻原型样例”。

---

## 项目2：ai-research-brief-generator

### 当前状态
- 已完成项目定位与能力目标定义
- 已有简历可讲版本
- 还缺最小输入输出样例

### 最小样例建议
#### 输入
- 3~5 条行业资讯 / 网页摘要 / 竞品动态

#### 输出
- 一页研究简报：
  - 主题
  - 关键信息
  - 趋势判断
  - 风险点
  - 建议动作

### 本轮最低交付
- [ ] 一组样例输入
- [ ] 一份固定输出模板
- [ ] 一份示例研究简报

---

## 项目3：ai-workflow-opportunity-scanner

### 当前状态
- 已完成项目定位与价值主张
- 已有机会评分方向
- 还缺一版可展示评分样例

### 最小样例建议
#### 输入
- 一个业务流程描述
- 现有痛点清单
- 人工重复环节

#### 输出
- 机会点评分表：
  - 价值
  - 可行性
  - 实施成本
  - 风险
  - 推荐优先级

### 本轮最低交付
- [ ] 一版评分维度说明
- [ ] 一个流程输入样例
- [ ] 一个输出评分样例

---

## 交付原则
- 不追求现在就做成完整系统
- 先保证“面试能展示、有东西可看、能体现思路”
- 优先做轻原型，而不是继续堆概念文案
