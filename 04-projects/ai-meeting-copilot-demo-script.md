# ai-meeting-copilot Demo Script

## 目标
用于 1~2 分钟内快速演示项目价值，让面试官看懂：
- 它解决什么问题
- 为什么不是普通摘要工具
- 当前已经做到哪一步

---

## 演示顺序

### 1. 先讲问题（10~15 秒）
很多团队开完会以后，信息分散在聊天记录、纪要和个人笔记里，导致会后最容易断的是行动项跟进。这个项目要解决的不是“记下来”，而是“跟下去”。

### 2. 展示原始会议输入（15 秒）
先展示一段相对真实的会议原始内容，强调几个痛点：
- 决策点散
- 责任人不清
- 截止时间容易漏
- 后续风险没人持续追踪

### 3. 展示结构化结果（20 秒）
系统把原始内容拆成：
- 会议结论
- 行动项
- 风险
- 下一步

这里重点强调：AI 的价值不是“写得更像纪要”，而是把可执行内容提取出来。

### 4. 展示行动项管理（20 秒）
再展示行动项视图，说明每个行动项都有：
- 内容
- 责任人
- 截止时间
- 状态（todo / in_progress / blocked / done）

这里要强调：项目真正的价值在于把会议输出接进执行流程。

### 5. 展示历史视图 / 仪表盘（15 秒）
展示历史会议或待跟进视图，说明这个项目不是一次性生成器，而是支持回看和持续跟进的轻量系统。

### 6. 收口（10~15 秒）
这个项目最重要的地方，是把“会议纪要”从信息沉淀升级成了行动闭环。当前已经完成可演示原型，下一步会继续打磨提醒机制和演示数据，把它收口成面试可展示的 MVP。

---

## 讲解关键词
- 不是摘要工具
- 是行动项闭环工具
- 从信息沉淀升级到执行系统
- 强调可追踪、可回看、可复盘
