# 3个月学习计划（执行版）

> 目标岗位：AI研究助理（业务流程优化 / 协同效率方向）
> 
> 执行原则：每周有产出、每月有可展示成果、每4周一次复盘。

---

## 总体里程碑

- **M1（第1-4周）**：打牢AI应用基础 + 完成项目1可展示MVP
- **M2（第5-8周）**：强化研究输出 + 完成项目2最小可用版
- **M3（第9-12周）**：强化业务AI化方案能力 + 项目3方案成型 + 面试材料闭环

---

## 每周固定节奏（建议）

- **周一**：本周目标拆解 + 任务排期（1小时）
- **周二~周四**：主线学习 + 项目开发（每天2~3小时）
- **周五**：整理本周输出（文档/代码/复盘，1~2小时）
- **周六**：模拟面试 or 项目讲述演练（1小时）
- **周日**：轻学习 + 下周准备（30分钟）

---

## 第1个月（第1-4周）
### 月目标
1. 建立 LLM / Prompt / RAG / Agent 的可落地认知
2. 完成 `ai-meeting-copilot` 可展示MVP
3. 打通简历项目描述与面试讲述版本

### 第1周：基础框架周
- 学习：
  - LLM能力边界、Prompt基础、RAG基本流程
  - Agent 与 Workflow 的区别（何时用，何时不用）
- 项目：
  - 梳理 `ai-meeting-copilot` 当前架构和待补清单
  - 固化会议输入→纪要→行动项→状态跟进主流程
- 产出：
  - 《AI应用基础框架笔记》1篇
  - 项目待办列表 v1（按优先级）

### 第2周：RAG与结构化输出周
- 学习：
  - 文档切分、检索策略、召回质量评估（precision/recall意识）
- 项目：
  - 优化纪要模板（结论/风险/行动项/责任人/截止时间）
  - 增加历史会议检索视图可读性
- 产出：
  - 《会议纪要结构化模板说明》
  - 项目演示脚本 v1（2分钟）

### 第3周：Agent/流程编排周
- 学习：
  - 单Agent与多步骤流程编排
  - 任务分解与状态管理基本设计
- 项目：
  - 行动项状态流转（待办/进行中/阻塞/完成）完善
  - 仪表盘字段补齐（总行动项、完成率、超期项）
- 产出：
  - 《行动项状态流转设计说明》
  - MVP展示截图/录屏素材

### 第4周：M1封版与求职对齐周
- 学习：
  - 岗位JD对齐：将技术点映射为业务价值表达
- 项目：
  - 完成项目1 MVP封版（演示路径可跑通）
- 求职材料：
  - 更新简历项目段落
  - 更新项目60~90秒讲稿
- 产出：
  - M1复盘（做了什么/效果/问题/下月改进）

- 产出：
  - 项目2 Demo v1

### 第8周：M2封版与表达强化周
- 项目：

- 求职材料：

- 产出：

  - 周报4：月度研究汇总

### 月目标
1. 强化业务流程AI化能力
2. 完成 `ai-workflow-opportunity-scanner` 方案与示例
3. 完成面试问答、STAR案例、项目讲稿闭环

### 第9周：流程诊断框架周
- 学习：

  - 三个项目统一整理成面试展示包
- 投递：
  - 重点岗位集中投递 + 跟进节奏执行
- 产出：
  - M3复盘 + 3个月总复盘

## 每日最小行动清单（可执行）
- 至少完成以下 3 项中的 2 项：
  1. 学习笔记 30 分钟（必须有文字输出）
  2. 项目推进 60~90 分钟（必须有可见改动）
  3. 求职材料优化 30 分钟（简历/讲稿/问答）

---

## 验收标准（3个月结束时）
- 项目：
  - 项目1有可展示MVP
  - 项目2有可运行最小版本
  - 项目3有完整方案+案例模板
- 材料：
  - 简历定制版稳定可投
  - 三项目60~90秒讲稿完成
  - STAR案例≥5，高频问答包可用
- 过程：
  - 每周至少1篇研究/复盘输出
  - 每月1次阶段复盘（问题+改进）
