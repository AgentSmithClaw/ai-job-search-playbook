# 岗位匹配规则

## 一、优先匹配的岗位关键词
优先关注以下方向：
- AI研究助理
- AI应用研究
- AI产品助理 / AI产品运营
- AI解决方案助理
- 智能化 / 数字化 / 效率工具相关岗位
- 知识管理 / 工作流优化 / 自动化相关岗位

## 二、优先匹配的能力要求
如果岗位描述中出现以下任意 3 项以上，可优先投递：
- 需求分析
- 流程梳理
- 项目推进
- 跨部门协同
- 自动化 / 效率提升
- 文档整理 / 研究输出
- AI 工具调研 / 应用落地
- 试点验证 / 方案评估

## 三、谨慎匹配的岗位
以下岗位只有在描述明显偏应用层时才考虑：
- 算法工程师
- 机器学习工程师
- 大模型训练 / 微调工程师
- 计算机视觉 / 推荐算法 / 搜索排序工程师

如果 JD 强调以下内容占主导，则降级处理：
- 深度学习底层研发
- 大规模训练经验
- 论文 / 顶会 / 学术成果
- 复杂模型部署优化
- 强算法竞赛背景

## 四、加分匹配信号
若岗位同时满足以下条件，优先级上调：
- 强调业务理解而非纯算法
- 提到流程优化、知识管理、协同效率、内部工具
- 接受复合背景候选人
- 岗位职责包含研究、整理、输出、试点推进
- 公司处于 AI 应用探索阶段，需要从 0 到 1 建机制

## 五、投递分级
### A 级：优先投递
- 岗位方向与 AI 应用研究、流程优化、协同提效高度一致
- 用户当前经历能直接转译
- 简历与项目能较好承接

### B 级：可投递
- 岗位和用户能力有部分重叠
- 需要通过简历优化和面试话术增强匹配度

### C 级：选择性投递
- 岗位有少量相关性，但核心要求偏底层技术或偏销售/纯运营

### D 级：不建议投递
- 以底层算法研发为绝对核心
- 与现有经验、短期补齐路径差距过大

## 六、筛选判断模板
看到一个新岗位时，按下面 5 个问题快速判断：
1. 它更偏业务应用还是偏底层算法？
2. 是否需要大量深度模型研发经验？
3. 是否看重流程分析、研究输出、协同推进？
4. 我的项目和经历能否自然接上？
5. 我能否在简历和面试里讲出明确匹配逻辑？

如果第 1、3、4、5 题答案大多为“是”，可进入投递池。
