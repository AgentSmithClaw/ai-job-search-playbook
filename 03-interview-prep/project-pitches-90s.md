# 项目讲述稿（60~90秒）

## 项目1：ai-meeting-copilot
这个项目解决的是团队会后执行断档问题。很多团队有会议记录，但没有可追踪的行动项。我把目标定义为“纪要结构化+行动项闭环”，并设计了从输入会议内容到输出结论、责任人、截止时间和状态的完整链路。技术上采用 FastAPI + SQLite + 轻前端原型，确保可快速迭代。当前已经支持历史会议查询、待办视图和仪表盘展示。这个项目的价值不在于生成摘要本身，而是把信息沉淀变成执行系统。下一步会继续打磨提醒机制和演示数据，形成稳定MVP。

## 项目2：ai-research-brief-generator
这个项目面向AI研究助理高频工作：把分散信息转成结构化研究简报。核心目标是让输出可以直接用于汇报和决策，而不是信息堆叠。方案上我会把流程拆成信息采集、清洗去重、主题归类、摘要生成和建议输出，最终形成统一模板（结论、证据、风险、建议）。当前已完成项目定位与框架设计，下一步是打通最小链路：输入资讯，输出可读简报，并增加来源依据，提升可信度。

## 项目3：ai-workflow-opportunity-scanner
这个项目解决企业“知道要做AI但不知道先做什么”的问题。我设计了一个流程机会扫描框架，用重复度、规则性、数据可得性和价值可行性做评分，把候选场景排序并给出试点建议。它的价值是帮助团队从大而空的讨论，转向可执行的落地路径。当前已完成定位与评估框架规划，下一步会补评分权重和2~3个案例模板，用于面试展示“从业务问题到AI试点”的完整能力链条。
